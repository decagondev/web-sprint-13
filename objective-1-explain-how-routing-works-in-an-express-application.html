Express Routing

Understanding How Routing Works in Express Applications

What is Express Routing?

Routing refers to how an application's endpoints (URIs) respond to client requests. Express provides a robust routing system that allows you to:

  • Handle different HTTP methods (GET, POST, PUT, DELETE)
  • Process requests at specific URLs
  • Send appropriate responses back to clients
  • Organize code based on routes and HTTP methods

Basic HTTP Methods

GET

Retrieve data from the server

server.get('/about', (req, res) => { res.status(200).send('

About Us

'); });

POST

Create new data on the server

server.post('/hobbits', (req, res) => { res.status(201).json({ url: '/hobbits', operation: 'POST' }); });

Advanced HTTP Methods

PUT

Update existing data on the server

server.put('/hobbits', (req, res) => { res.status(200).json({ url: '/hobbits', operation: 'PUT' }); });

DELETE

Remove data from the server

server.delete('/hobbits', (req, res) => { res.status(204).send(); });

Common HTTP Status Codes

200 OK

Successful GET/PUT requests

201 Created

Successful POST requests

204 No Content

Successful DELETE requests

404 Not Found

Resource not found

Follow Along: Building a Complete API

Let's create a complete set of CRUD endpoints for a hobbits resource. Each endpoint will handle different HTTP methods on the same URL. 

// GET request handler
server.get('/hobbits', (req, res) => {
    res.status(200).json({ url: '/hobbits', operation: 'GET' });
});

// POST request handler
server.post('/hobbits', (req, res) => {
    res.status(201).json({ url: '/hobbits', operation: 'POST' });
});

// PUT request handler
server.put('/hobbits', (req, res) => {
    res.status(200).json({ url: '/hobbits', operation: 'PUT' });
});

// DELETE request handler
server.delete('/hobbits', (req, res) => {
    res.status(204).send();
});

Important Notes

  • Each HTTP method can use the same URL but will trigger different handlers
  • Different status codes are used for different operations
  • Response format can be HTML or JSON depending on your needs
  • Later, we'll learn to handle request data using tools like Postman

Testing Your Routes

Browser Testing (GET only)

Navigate to /about and /contact in your browser to test GET routes.

Postman Testing (All Methods)

Use Postman to test POST, PUT, and DELETE routes. We'll learn more about this in upcoming lessons.
