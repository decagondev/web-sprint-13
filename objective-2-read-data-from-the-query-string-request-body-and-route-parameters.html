Reading Data from Requests in Express

Understanding Query Strings, Request Body, and Route Parameters

Introduction Video

Route Parameters

Dynamic segments in URLs that capture values from the request path.

/users/:id

Query String

Key-value pairs appended to the URL after a question mark.

?sort=name&order=desc

Request Body

Data sent in the request payload, typically for POST/PUT requests.

req.body.data

Route Parameters

Route parameters are dynamic segments in the URL pattern that capture values from the actual request URL. They are prefixed with a colon (:) in the route definition.

Key Points:

  • Values are accessed via req.params
  • Parameters are always strings
  • Multiple parameters are supported
  • Common for RESTful resource identification
// Route Definition
app.get('/users/:id/posts/:postId', (req, res) => {
    const userId = req.params.id;
    const postId = req.params.postId;
    // Access both parameters
});

Query String Parameters

Query strings provide a way to pass data to the server through the URL. They start with a question mark (?) and consist of key-value pairs.

Common Use Cases:

  • Pagination (?page=2&limit=10)
  • Filtering (?category=books)
  • Sorting (?sort=price&order=desc)
  • Search (?q=search+term)

Important Notes:

  • Access via req.query
  • Case-sensitive parameter names
  • Arrays possible with repeated keys
  • Values are strings by default
// Handling query parameters
app.get('/api/products', (req, res) => {
    const { sort, order, page = 1, limit = 10 } = req.query;
    // Use the parameters for filtering/sorting
});

Request Body

The request body is used to send data to the server, typically in POST, PUT, or PATCH requests. Express needs middleware to parse the body data.

Important Setup

Don't forget to add the body-parser middleware:

app.use(express.json());

Request Example:

POST /api/users
Content-Type: application/json

{
  "name": "John Doe",
  "email": "john@example.com",
  "age": 30
}

Handler Example:

app.post('/api/users', (req, res) => {
  const { name, email, age } = req.body;
  // Create new user
});

Follow Along: Complete Example

Let's create a complete example that uses all three methods of reading request data:

const express = require('express');
const app = express();

// Middleware for parsing JSON bodies
app.use(express.json());

// GET users with query parameters for filtering and sorting
app.get('/api/users', (req, res) => {
    const { sort, order, page = 1, limit = 10 } = req.query;
    // Use parameters to filter and sort users
});

// GET specific user by ID (route parameter)
app.get('/api/users/:id', (req, res) => {
    const userId = req.params.id;
    // Find user by ID
});

// POST new user (request body)
app.post('/api/users', (req, res) => {
    const { name, email, age } = req.body;
    // Create new user with data
});

// PUT update user (combination of route param and body)
app.put('/api/users/:id', (req, res) => {
    const userId = req.params.id;
    const updates = req.body;
    // Update user using both pieces of data
});
