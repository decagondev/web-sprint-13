Building Your First API

We'll create a simple Web API that responds to HTTP requests. This will serve as your foundation for building more complex APIs in the future.

Setup Steps

  1. Clone the node-express-mini repository
  2. Navigate to the project folder using cd
  3. Install dependencies with npm install
  4. Create index.js in the root directory

Basic Server Setup

const express = require('express');
const server = express();

server.get('/', (req, res) => {
    res.send('Hello World');
});

server.listen(8000, () => {
    console.log('API running on port 8000');
});

Key Concepts

  • Express application instance
  • Route handlers
  • Request/Response objects
  • Server listening

Important Notes

  • Server runs until stopped
  • Handles all incoming requests
  • Routes define behavior
  • Port must be available

Working with JSON Responses

Modern APIs typically respond with JSON data. Let's create an endpoint that returns a list of hobbits: 

server.get('/hobbits', (req, res) => {
    const hobbits = [
        { id: 1, name: 'Samwise Gamgee' },
        { id: 2, name: 'Frodo Baggins' },
    ];
    res.status(200).json(hobbits);
});

HTTP Status Codes

Always send appropriate status codes with your responses:

  • 200: OK (successful GET/PUT)
  • 201: Created (successful POST)
  • 204: No Content (successful DELETE)
  • 400: Bad Request
  • 404: Not Found
  • 500: Server Error

API Best Practices

Error Handling

  • Use try-catch blocks
  • Send proper status codes
  • Include error messages
  • Log errors server-side

Security

  • Validate input data
  • Sanitize responses
  • Use HTTPS
  • Implement rate limiting

Performance

  • Implement caching
  • Compress responses
  • Optimize queries
  • Use async/await
