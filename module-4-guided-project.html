Important Note About Deployment Options

This Guided Project demonstrates deploying an Express application to both Heroku and Render.com. While Heroku has traditionally been a popular choice, as of November 2022 Heroku does not provide a totally free tier. As an alternative, we recommend using Render.com which offers a generous free tier perfect for personal projects. We'll cover both platforms so you can choose the one that best fits your needs.

Project Overview

In this guided project, you will learn:

  • How to prepare an Express application for deployment
  • Understanding environment variables and configuration
  • Best practices for deployment security
  • Continuous deployment setup
  • Testing deployed applications

Deployment Steps

Preparing for Heroku Deployment

  1. Ensure your package.json has a "start" script: 
    "scripts": {
    "start": "node index.js"
}
  2. Use environment variables for configuration: 
    const port = process.env.PORT || 5000;
  3. Create a Procfile (optional): 
    web: npm start
  4. Initialize Git repository and commit changes

Deploying to Heroku

  1. Create a Heroku account
  2. Install Heroku CLI
  3. Login to Heroku: heroku login
  4. Create Heroku app: heroku create
  5. Push to Heroku: git push heroku main
  6. Set environment variables: heroku config:set KEY=VALUE

Preparing for Render.com Deployment

  1. Ensure your package.json has a "start" script: 
    "scripts": {
    "start": "node index.js"
}
  2. Use environment variables for configuration: 
    const port = process.env.PORT || 5000;
  3. Push your code to a GitHub repository

Deploying to Render.com

  1. Create a Render.com account
  2. Click "New +" and select "Web Service"
  3. Connect your GitHub repository
  4. Configure your service:
    • Name: Your service name
    • Environment: Node
    • Build Command: npm install
    • Start Command: npm start
  5. Set environment variables in the Render dashboard
  6. Click "Create Web Service"

Bonus: Render.com offers automatic deployments whenever you push to your GitHub repository, and their free tier includes SSL certificates, automatic builds, and more!
