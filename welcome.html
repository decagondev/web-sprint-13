Welcome to Sprint 13!

This week we'll learn how to build web services based on the REpresentational State Transfer (REST) architectural style.

We'll learn about Node.js, a runtime environment used to execute JavaScript code outside the browser. We will also learn about the popular Express framework that makes it easy to write web applications and services on top of Node.js.

Review Learning Resources and the Unit Syllabus before you begin

Module 1

Introduction to Node.js and Express

We will introduce Node.js and Express. Discover what Node.js is and what it can and cannot do. Explore Express and look at why it's a helpful tool to use. Learn how to test our Web API using Postman manually. Create our first CRUD Web API.

Module 2

Server-Side Routing with Express

Node's HTTP server uses a single request handler function for all requests. Express provides routing to break the application into different request handlers for each URL + HTTP method combination. We'll cover REST architecture and use Express Routers to organize our code.

Module 3

Express Middleware

Express is a minimalist framework that can be extended using middleware. We'll learn how middleware adds functionality to our application and provides modularity to our code. Tasks like authentication and logging are commonly handled by middleware.

Module 4

Web Deployment and Best Practices

The time has come to show your work to the world. We'll learn how to make our API deployable by extracting configuration details and preparing for deployment to Heroku and other similar services.
